# 카테고리별 웨딩 준비 질의응답 가이드

## 목적

사용자가 `드레스 준비 미흡한 점 있나`처럼 물으면 BFF가 저장된 준비 데이터와 카테고리별 Markdown 가이드를 비교해 미확인 항목을 안내하고 다음 질문을 생성한다.

## 가이드 조회

`GET /api/chat/guides/{category}`

- category: `hall`, `studio`, `dress`, `makeup`
- 대문자 별칭도 허용한다.
- 지원하지 않는 카테고리는 404를 반환한다.

```json
{
  "category": "dress",
  "content": "# 드레스 준비 점검 가이드\n..."
}
```

## BFF 처리 순서

1. 사용자 발화를 `query_readiness` 의도로 분류하고 카테고리를 추출한다.
2. 카테고리가 없으면 웨딩홀·스튜디오·드레스·메이크업 중 무엇을 점검할지 질문한다.
3. 백엔드에서 가이드와 현재 카드 상세를 조회한다.
4. LLM 입력에 가이드 원문, 카드·납부·일정·메모, 관련 채팅 내역을 함께 전달한다.
5. 확인된 내용은 반복하지 않고 미확인 항목을 중요도 순으로 최대 세 개 안내한다.
6. 응답 마지막에는 가장 중요한 후속 질문 하나만 둔다.
7. 후속 답변에서 날짜·금액·업체 정보가 나오면 기존 챗봇 추출·임시 카드 흐름으로 넘긴다.

## 프롬프트 안전 규칙

- 저장된 데이터에 없는 항목은 `하지 않음`이 아니라 `미확인`으로 표현한다.
- Markdown 가이드에 없는 계약 조건을 지어내지 않는다.
- 업체별 실제 계약서나 안내가 가이드보다 우선한다.
- 한 번에 모든 질문을 나열하지 않는다.
- 일정과 금액은 답변 문장에서 추측해 채우지 않는다.

## 예시

사용자: `드레스 준비 미흡한 점 있나`

응답: `드레스 업체와 계약 금액은 확인됐지만 최종 피팅 일정, 추가금, 수령 방법은 미확인이에요. 먼저 다음 피팅 날짜를 잡으셨나요?`
