# 문서 분석 결과 저장 API

## 역할 분리

- `POST /api/cards/{cardId}/analyze`와 LLM 호출은 BFF가 담당한다.
- 백엔드는 사용자가 수락한 일정과 납부 제안만 검증하고 저장한다.
- todo 제안은 메모 체크리스트가 원본이므로 별도 엔티티로 저장하지 않는다.

## 분석 결과 반영

`POST /api/cards/{cardId}/analyze/apply`

```json
{
  "events": [
    {
      "title": "드레스 가봉",
      "date": "2026-07-12",
      "time": "14:00",
      "quote": "가봉 7/12 오후 2시"
    }
  ],
  "payments": [
    {
      "label": "잔금",
      "amount": 360000,
      "dueDate": null,
      "quote": "잔금 36만원"
    }
  ]
}
```

- 응답은 `GET /api/cards/{cardId}`와 같은 카드 상세 형태다.
- 일정 시간이 없으면 `time`은 `null`로 보낸다.
- 납부일이 없으면 카드가 속한 결혼식의 본식일을 사용한다. 본식일도 없으면 400을 반환한다.
- 같은 카드, 제목, 일시의 일정은 다시 생성하지 않는다.
- 같은 라벨의 미납 항목이 있으면 새 항목을 만들지 않고 `quote`를 메모에 반영한다.
- 한 요청에는 일정과 납부를 합쳐 최대 20개까지 보낼 수 있다.
- 카드 소유자가 아니면 404를 반환한다.

## BFF 연결

프론트가 보내는 `{ "accepted": ["s1", "s2"] }`는 BFF 내부 계약으로 유지한다. BFF는 보관 중인 제안에서 수락된 ID를 찾아 `event`와 `payment`를 위 요청의 `events`, `payments`로 나눈 뒤 백엔드를 호출한다.
